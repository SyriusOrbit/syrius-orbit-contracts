SyriusOrbit API overview.

SyriusOrbit is a spatial information and scheduling service platform for indoor Autonomous Mobile Robots (AMRs). Its API surface is organized into two suites: the SyriusOrbit Spatial API, which uses OpenAPI for spatial data and geospatial services aligned with the OGC API family, and the SyriusOrbit Fleet API, which uses AsyncAPI for robot communication, scheduling, and fleet messaging aligned with VDA 5050 and related robot interoperability standards.

SyriusOrbit Spatial API

The Spatial API focuses on spatial data and geospatial services. It follows the OpenAPI format and is organized around OGC API concepts for resource-oriented access.

  • Spatial resources and geospatial query patterns
  • Maps, coverages, routes, and moving features
  • Documented as a REST-oriented API contract

SyriusOrbit Fleet API

The Fleet API focuses on robot communication, task exchange, and fleet messaging. It follows the AsyncAPI format and is organized around VDA5050-based interaction flows.

  • Fleet state, order exchange, and event channels
  • Message-based communication patterns
  • Documented as an asynchronous API contract
